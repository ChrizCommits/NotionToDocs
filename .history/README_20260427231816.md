# RecipeSync Automation (n8n)

Automated synchronization system that turns a Notion recipe database into a single structured Google Document.
Manage recipes in Notion, while the document stays automatically updated and provides a clean, AI-ready source for tools like Gemini.

**Tip:** *Include available ingredients (e.g. a short list or a fridge screenshot) when using Gemini for more relevant recipe suggestions.*

## ⚙️ Overview
This project consists of two workflows:

* **Initial Sync** → builds the full document from scratch (updates daily*)
* **Incremental Sync** → updates only changed recipes (updates every minute*)

* *can be adjusted easily in the nodes

The system ensures:

* consistent structure using markers
* efficient updates via incremental processing
* automatic cleanup of deleted entries

---

### Key Concepts
**Markers**
Define sections in the document to safely replace content

**Incremental Updates**
Only modified recipes are processed

**Cleanup Logic**
Deleted recipes are removed from the document

## 🔄 Workflows
### Initial Sync
```text
→ Get recipes from Notion
→ Build new recipe content
→ Replace old with new content
```

### Incremental Sync
```text
PREPARING AND GATHERING INFO
→ Fetch sync states & recipes 

CHECK & INSERT MARKERS
→ If marker not found → Insert
→ If marker found → Forward

CREATE NEW CONTENT
→ Get recipe data and wrap markers

CREATE OLD CONTENT
→ Get content from marker to marker

UPDATE DOCUMENT
→ Replace old and new content

CLEANUP DELETED RECIPES
→ Extract recipe IDs & sections
→ Identify already deleted recipes
→ Delete recipes in the document
```

## 📊 Scaling and Performance
Tested with:
```text
500 recipes (fully populated)
```

Important:
The incremental workflow is not designed for mass updates and just updates recipes really quick.
Large changes (> 50 recipe changes per minute) should be handled by the daily full sync.

## 🛠️ Setup

1. Import workflows:
   * initial-setup.json  
   * incremental-sync.json  

2. Prepare Notion:
   * Create a database for your recipes  
   * Create an additional database called `RecipeSync`  
   * Add a text property `lastSync`  
   *This database is used to store the last sync timestamp for incremental updates*

3. Connect credentials:
   * Notion  
   * Google Document API  

4. Run:
   * Execute Initial Sync once to build the document  

5. Activate workflows:
   * Activate Initial Sync (for daily runs)  
   * Activate Incremental Sync (for frequent updates)
